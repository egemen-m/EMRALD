A Guide to EMRALD

What is EMRALD?

EMRALD (Event Modeling Risk Assessment using Linked Diagrams) is a user interface used to facilitate creating and visualizing PRA models. EMRALD can be couple with other analysis applications such as physics based simulations - this includes one-way communication for most applications and two-way loose coupling for customizable applications.

Project

What is a project?

A project includes all elements of a PRA model. Within a project you have several types of components - diagrams, logic trees, external simulations, action, events, variables and states. Within the EMRALD site, only one project can be open at a time. Most project actions can accessed by clicking the “Project” button on the upper menu bar.

How do you create a new project?

In the top left corner of the upper menu bar, click on “Project”. Select “New” from the dropdown menu. Then fill out the dialog box and press “OK”.

How do you save a project?

Saving a project (i.e. exporting a JSON containing the project information) can be done two different ways.

Option 1: In the top left corner of the upper menu bar, click on “Project”. Select “Save” from the dropdown menu.

Option 2: Open any Diagram and right click anywhere on the main screen (not on any element). Select "Save".

Diagram

What is a Diagram?

Each diagram represents a particular piece of the model and the various conditions or states that this piece of the model can be in. These pieces correlate to aspects of traditional PRA modeling and range from small-scale components to a large scope, overall plant response and design. A diagram contains multiple states with the events that can occur, actions that can be executed and variables used. These all define how the simulation may sift through the diagram over time.

How do you create a new diagram?

On the left sidebar, right click on the Diagrams label. Click "New Diagram", and fill out the dialog box and press "OK". It will then open an empty diagram. The diagram should also appear in the sidebar menu.

How do you edit a diagram?

Option 1: On the left sidebar, click on the Diagrams label to expand the menu. Click on the folder corresponding to the type of diagram you want to edit

Option 2: Open any Diagram and right click anywhere on the main screen (not on any element). Select "Save".

Logic Tree

What is a Logic Tree?

Logic trees use standard boolean gates to map and evaluate the behaviour of components. These trees use the assigned Boolean value for the state a component is currently, to solve for the top value of the tree.

How do you create a new logic tree?

On the left sidebar, right click on the Logic Tree label. Click "New Logic Tree", and fill out the dialog box and press "OK". It will then open an empty logic tree editor. The logic tree should also appear in the sidebar menu.

How do you edit a logic tree?

1) Opening the logic tree: On the left sidebar, click on the Logic Tree label to expand the menu. Double click on the tree you want to edit

2) Changing gate properties: Click on the edit button on the bottom right of the gate and change the settings in the dialog box.

How do you add logic gates?

Option 1: On the bottom right of the desired parent gate, click on the plus icon. Fill out the dialog box and click okay.

Option 2: drag and drop a logic gate from the toolbar onto the desired parent gate. Note: component diagrams can not have child gates.

How do you add diagrams to the tree?

Drag and drop (component diagrams only) from the sidebar onto the desired parent gate.

Actions

What is an Action?

Actions change the properties or cause movement though a model during a simulation run. The following are the main types of actions but with the expandable design of EMRALD additional or customized actions can easily be added.

Transition – Indicates a state or set of states that the simulation will process and be made current.

Change Variable – Changes the value of a variable according to a user defined script.

Execute Application – Runs user defined scripts to both execute an external piece of code and process the results to direct state changes.

External Sim Message – Sends a message to an external code through coupled communication messaging protocol.

How do you edit an action?

Option 1: On the left sidebar, click on the tab (All, Global or Local) where your action is located. If you are not sure which tab, click on the All tab. Right click on the action and click "Edit properties.

Option 2: Open the Diagram containing the action and locate the action. Right click on the action and click "Edit Action".

How do you add an action?

Option 1: On the left sidebar, click on the tab (All, Global or Local) where you want your action to be located. Right click on the action label and click "Add Action". Fill out the dialog box and press "OK".

Option 2: Open the Diagram you would like your action to be in. Locate the State you would like to add the action to and either right click on the immediate action label or on the event you would like to add it to. Click "New Action" and fill out the dialog box and press "OK".

Events

What is an Event?

Events monitor for specified criteria and can have one or more actions that are executed when that criteria is met. There are two categories of events; time based, and conditional, with different types in each category.

Time Based:
Timer – Monitors the current time against the user specified time.
Failure Rate – Samples a given probability of failure to determine the time of this event.
Distribution – Samples a distribution according to user specified parameters for the event time.

Conditional: State Change – Monitors when entering or exiting a specified state to determine event execution.
Component Logic Tree – Evaluates a logic model of component diagrams to determine event execution.
Variable Condition Event – Monitors variable values with user-defined code to determine event execution.
External Sim Event – Links to an external simulation flag to determine event execution.

Actions executed by an event are state dependent, meaning that if the same event is in more than one state, the event can trigger different actions for each state. Multiple actions can also be assigned to a single event and each of these actions is executed if the event occurs. Finally, each event in a state has a flag that indicates if this state is to be exited if the event occurs.

How do you edit an event?

Option 1: On the left sidebar, click on the tab (All, Global or Local) where your event is located. If you are not sure which tab, click on the All tab. Right click on the event and click "Edit properties".

Option 2: Open the Diagram containing the event and locate the event. Right click on the event and click "Edit Event".

How do you add an event?

Option 1: On the left sidebar, click on the tab (All, Global or Local) where you want your event to be located. Right click on the event label and click "Add Event". Fill out the dialog box and press "OK".

Option 2: Open the Diagram you would like your event to be in. Locate the State you would like to add the event to and right click on the events label. Click "New Event" and fill out the dialog box and press "OK".

Variables

What is a Variable?

Variables define a value that can be evaluated or modified by user defied scripts in some events and actions. External variables are linked to an external code and can be modified or evaluated by either.

How do you create a new variable?

On the left sidebar, click on the tab (All or Global) where you want your variable to be located. Right click on the variable label and click "Add Variable". Fill out the dialog box and press "OK".

How do you edit a variable?

On the left sidebar, click on the tab (All or Global) where your variable is located. If you are not sure which tab, click on the All tab. Right click on the variable and click "Edit properties".

States

What is a State?

States are a logical representation for a current condition in a diagram. Each state has or can have the following attributes:

Type – Start, Standard, Key State, or Terminal, which establishes simulation behaviour when setting up, running or post-processing a simulation run.

Immediate Actions – These actions are executed when entering the state.

Event Actions – Items to monitor when in this state which can then trigger one or more actions.

How do you create a new state?

On the left sidebar, click on the tab (All or Local) where you want your state to be located. Right click on the state label and click "Add State". Fill out the dialog box and press "OK".

How do you edit a state?

Option 1: On the left sidebar, click on the tab (All or Local) where your state is located. If you are not sure which tab, click on the All tab. Right click on the state and click "Edit properties".

Option 2: Open the Diagram you would like your state to be in. Right click anywhere on the screen (not on any objects). Click "New State" and fill out the dialog box and press "OK".
